Git Commands

  1. GIT - Global Information Tracker
  2. git init - Initialize a new git repository
  3. git clone [url] - Clone a repository from a URL
  4. git status - Check the status of the repository
  5. git add [file] - Stage a file for commit
  6. git commit -m "message" - Commit staged changes with a message
  7. git push - Push committed changes to a remote repository
  8. git pull - Pull changes from a remote repository | git fetch + git merge
  9. git branch - List, create, or delete branches
  10. git checkout [branch] - Switch to a different branch
  11. git merge [branch] - Merge a branch into the current branch
  12. git log - View the commit history
  13. git diff - Show changes between commits, commit and working tree, etc.
  14. git remote - Manage remote repository connections
  15. git fetch - Download objects and refs from another repository
  16. git reset [file] - Unstage a file
  17. git rm [file] - Remove a file from the working directory and stage the removal
  18. git stash - Stash changes in a dirty working directory
  19. git tag [name] - Create a tag for a specific commit
  20. git config - Configure git settings
  21. git show [commit] - Show various types of objects
  22. git blame [file] - Show what revision and author last modified each line of a file
  23. git cherry-pick [commit] - Apply the changes introduced by some existing commits
  24. git rebase [branch] - Reapply commits on top of another base tip
  25. git archive - Create an archive of files from a named tree

Revert Operation before push

To revert changes before pushing to a remote repository, you can use the following commands:

git reset --hard [commit id] : Reset the current branch to the previous commit, discarding all changes

git reset --soft [commit id] : Reset the current branch to the previous commit, keeping changes in the working directory

git reset --mixed [commit id] : Which is default | Reset the current branch to the previous commit, keeping changes in the staging area to unstage area

git revert [commit id] : Create a new commit that undoes the changes made in [commit id]

git reflog : Show a log of all actions (including resets , reverts and checkouts) that have affected the HEAD

Resolve Conflicts

When you encounter merge conflicts during a git merge or git pull, you need to resolve them manually. Here are the steps to resolve conflicts:

  1. Identify the files with conflicts. Git will mark the conflicted areas in the files.
  2. Open the conflicted files and look for the conflict markers (<<<<<<<, =======, >>>>>>>).
  3. Manually edit the files to resolve the conflicts by choosing one side or combining changes.
  4. After resolving the conflicts, stage the changes using git add [file].
  5. Finally, commit the changes with git commit.

git merge --abort : Abort the merge process and return to the pre-merge state

Stashing Changes

To temporarily save changes that are not ready to be committed, you can use the git stash command:

  1. Run git stash to save your changes and revert to the last commit.
  2. Run git stash -m "message" to stash changes with a message.
  3. Run git stash show to view the changes in the stash.
  4. Run git stash list to view all stashed changes.
  5. Run git stash apply [stash id] to reapply stashed changes.
  6. Run git stash pop to reapply the most recent stash and remove it from the stash list.
  7. Run git stash pop [stash id] to reapply a specific stash and remove it from the stash list.
  8. Run git stash drop [stash id] to delete a specific stash.
  9. Run git stash clear to delete all stashes.

Cherry Picking

Cherry picking is the process of selecting specific commits from one branch and applying them to another branch. Here are the steps to cherry-pick a commit:

  1. Identify the commit hash you want to cherry-pick using git log.
  2. Switch to the target branch where you want to apply the commit.
  3. Run git cherry-pick [commit hash] to apply the commit.
  4. Run git cherry-pick --abort to cancel the cherry-pick operation.
  5. If there are conflicts, resolve them manually, stage the changes using git add [file], and then run git cherry-pick --continue.

.gitignore uses

The .gitignore file is used to specify files and directories that should be ignored by Git. Here are some common uses of .gitignore:

  • Ignore temporary files created by your text editor or IDE.
  • Ignore build artifacts and compiled files.
  • Ignore log files and debug output.
  • Ignore operating system-specific files (e.g., .DS_Store on macOS).
  • Ignore dependency directories (e.g., node_modules for Node.js projects).
  • Ignore configuration files (e.g., .env) that contain sensitive information.
