Git Commands

  1. GIT - Global Information Tracker
  2. git init - Initialize a new git repository
  3. git clone [url] - Clone a repository from a URL
  4. git status - Check the status of the repository
  5. git add [file] - Stage a file for commit
  6. git commit -m "message" - Commit staged changes with a message
  7. git push - Push committed changes to a remote repository
  8. git pull - Pull changes from a remote repository | git fetch + git merge
  9. git branch - List, create, or delete branches
  10. git checkout [branch] - Switch to a different branch
  11. git merge [branch] - Merge a branch into the current branch
  12. git log - View the commit history
  13. git diff - Show changes between commits, commit and working tree, etc.
  14. git remote - Manage remote repository connections
  15. git fetch - Download objects and refs from another repository
  16. git reset [file] - Unstage a file
  17. git rm [file] - Remove a file from the working directory and stage the removal
  18. git stash - Stash changes in a dirty working directory
  19. git tag [name] - Create a tag for a specific commit
  20. git config - Configure git settings
  21. git show [commit] - Show various types of objects
  22. git blame [file] - Show what revision and author last modified each line of a file
  23. git cherry-pick [commit] - Apply the changes introduced by some existing commits
  24. git rebase [branch] - Reapply commits on top of another base tip
  25. git archive - Create an archive of files from a named tree

Revert Operation after push and before push

To revert changes after pushing to a remote repository, you can use the following commands:

git reset --hard [commit id] : Reset the current branch to the previous commit, discarding all changes

git reset --soft [commit id] : Reset the current branch to the previous commit, keeping changes in the working directory

git reset --mixed [commit id] : Which is default | Reset the current branch to the previous commit, keeping changes in the staging area to unstage area

git revert [commit id] : Create a new commit that undoes the changes made in [commit id]
